项目名称：PyTorch适配KML数学库的优化可行性定位与优化实现

项目内容：华为鲲鹏920系列处理器为arm架构，基础数学库KML为鲲鹏920平台提供上层软件适配，主要用于加速向量、矩阵运算以及浮点计算等数学运算，通过Neon指令优化、内联汇编、向量化等手段，显著提高计算性能。实习过程中，我主要负责PyTorch对接底层数学库KML_BLAS的优化分析与具体实现。由于在CPU或其他资源有限平台上，PyTorch的主要应用场景围绕模型部署与推理，因此优化可行性分析主要从模型推理过程的访存优化来切入。

1. 考虑到目标检测模型是端侧部署中最常见的应用，因此选取几类经典模型 ResNet50/101，MobileNet-v1/v2/v3，在ImageNet数据集上进行推理实测，通过perf和 PyTorch profiler 追踪神经网络模型推理过程的热点函数。可以发现，PyTorch调用KML_BLAS实现卷积算子的操作有较多的CPU占用。
2. 进一步定位，发现 tensor 的常用内存排布为 nchw/nhwc，与 kernel 对应的 feature map 中的数据不是连续分布的。如果 feature map 空间很大的话，这样不是按照通道顺序取数据，指针会不断来回地在内存空间移动，会造成较高的 cache miss。为了充分发挥指令集的并行化优势，进行通道维度的拆分。根据不同寄存器特性，可对 channel 维进行了 channels/4，channels /32 和 channels /64 等的拆分，转换为 nc4hw4 排布。
3. 在C方向上可并行，在H W方向上连续访问，进行通道拆分后，可配合neon进行加速。最终实测发现，在 feature map 的 H W < 512 且 channels > 16 时，卷积算子平均推理性能提升可达16%。
4. 获评“优秀实习生”。

**024.06-2024.09，参与华为鲲鹏计算“PyTorch对接KML_BLAS优化”项目，主要完成了以下工作：**

1. 性能分析： 针对端侧部署中常见的目标检测模型，在ImageNet数据集上进行推理测试。利用perf和PyTorch Profiler工具，追踪神经网络模型推理过程中的热点函数，发现PyTorch调用KML_BLAS实现卷积算子时CPU占用较高。 
2. 问题定位： 进一步分析发现，张量常用的内存排布格式（NCHW/NHWC）导致与卷积核对应的特征图数据在内存中非连续分布。对于大型特征图，非按通道顺序读取数据会导致指针在内存空间中频繁跳转，造成较高的缓存未命中（cache miss）。 
3. 优化实现： 为充分发挥指令集的并行优势，进行了通道维度的拆分。根据不同寄存器的特性，将通道维度拆分为channels/4、channels/32和channels/64等，转换为NC4HW4的内存排布方式。 
4. 加速验证： 在通道方向实现并行化，且在高度（H）和宽度（W）方向实现连续访问。 

解决的难点和问题： 

1. CPU占用高： 解决了PyTorch在ARM架构处理器上调用KML_BLAS实现卷积算子时CPU占用过高的问题。 
2. 缓存未命中： 减少了由于张量内存排布导致的非连续内存访问，降低了缓存未命中率，提高了数据访问效率。 

取得的成果： 

1. 性能提升： 结合Neon指令进行加速，当特征图的高度乘宽度（H×W）小于512且通道数大于16时，卷积算子的平均推理性能提升可达约18%。
